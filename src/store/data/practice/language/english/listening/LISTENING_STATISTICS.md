## 🔍 Exercise Structure Analysis

### Exercise Types Available

Each listening exercise contains:

1. **Multiple Choice Questions** (`multiple_choice`)
   - Typically 4-6 questions per exercise
   - 4 options per question (a, b, c, d)
   - Clear correct answers
   - Questions must be answerable from the audio content

2. **True/False Questions** (`trueFalse`)
   - Typically 4-6 statements per exercise
   - True/False answers
   - Explanations provided
   - Statements must be verifiable from the audio content

### Exercise Metadata

Each exercise includes:
- `id`: Unique identifier (sequential number)
- `title`: Exercise title (descriptive)
- `level`: CEFR level (A1-C2)
- `durationInMinutes`: Estimated duration
- `topic`: Topic category (e.g., "Daily Activities", "Shopping", "Travel")

### Required Files Per Exercise

For each complete listening exercise, you need:

1. **Audio File** (`.mp3` or `.m4a`)
   - Compressed, optimized for web
   - Clear audio quality

2. **Transcription** (format to be determined) - **MANDATORY**
   - ⚠️ **REQUIRED:** Every audio file MUST have a corresponding transcription
   - ❌ **Currently NOT stored** - Needs to be created for all existing and future audio files
   - Complete word-for-word text of the entire audio
   - Speaker identification if multiple speakers
   - Proper punctuation and capitalization
   - **Storage format:** To be decided (may be `.txt`, `.md`, or added to JSON)
   - **Rule:** No audio file can be submitted without its transcription

3. **Exercise Data** (added to `{level}.json`)
   - JSON object with all exercise information
   - Questions and options
   - Metadata

4. **Answer Key** (added to `{level}-answers.json`)
   - Correct answers
   - Explanations (optional but recommended)

## 🎙️ Audio File Preparation Guidelines

### ⚠️ CRITICAL: Audio Compression Requirements

**Before uploading audio files, they MUST be compressed to optimize file size and loading performance.**

#### Recommended Compression Settings:

1. **Format:** MP3 (preferred) or M4A
2. **Bitrate:** 64-128 kbps (sufficient for speech)
3. **Sample Rate:** 22.05 kHz or 44.1 kHz
4. **Channels:** Mono (for speech) or Stereo (if needed)
5. **Target File Size:** Keep files under 5MB when possible

#### Compression Tools:

- **Online Tools:**
  - CloudConvert (https://cloudconvert.com)
  - Online Audio Converter (https://online-audio-converter.com)
  - FreeConvert (https://www.freeconvert.com/audio-converter)

- **Desktop Software:**
  - Audacity (Free, open-source)
  - VLC Media Player (Free)
  - FFmpeg (Command-line, free)

#### Compression Steps (using Audacity):

1. Open audio file in Audacity
2. Go to **File → Export → Export as MP3**
3. Set **Bitrate** to 64-128 kbps
4. Set **Quality** to "Standard" or "High"
5. Click **Save**
6. Verify file size is reasonable (< 5MB for typical 5-10 minute audio)

#### File Naming Convention:

- Use descriptive, lowercase filenames with hyphens
- Format: `{number}-{descriptive-name}.mp3`
- Examples:
  - `11-daily-conversation-at-cafe.mp3`
  - `12-job-interview-practice.mp3`
  - `13-weather-forecast-news.mp3`
